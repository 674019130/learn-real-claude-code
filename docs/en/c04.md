# 100+ Slash Commands

> Commands are user-initiated UI actions; tools are model-initiated API actions — they look similar but live on opposite sides of the human/model boundary.

## Overview

Type `/compact` in a Claude Code session and the conversation history gets summarized. Type `/memory` and you see what the agent remembers between sessions. These are slash commands — and they're a completely different system from the tools the model uses internally.

The confusion between commands and tools is one of the most common conceptual stumbling blocks for developers building their own agents. Both look like "calling a function." Both produce effects. But they operate at different layers of the system and serve fundamentally different purposes.

`commands.ts` implements the full command registry: over 100 built-in commands covering conversation management, memory inspection, context control, file review, and user-defined extensions. This chapter walks through the registration pattern, the dispatch mechanism, and how commands relate to (and sometimes invoke) the tool system underneath.

## Commands vs Tools — The Fundamental Split

The distinction is about *who initiates the action*:

```
┌─────────────────────────────────────────────────────────┐
│                    User types /compact                  │
│                           │                             │
│                    Command Dispatcher                   │
│                    (commands.ts)                        │
│                           │                             │
│              Runs CompactCommand.handler()              │
│                           │                             │
│         May invoke tools internally if needed           │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│              Model decides to read a file               │
│                           │                             │
│                    Agent Loop (query.ts)                │
│                           │                             │
│         StreamingToolExecutor fires ReadFileTool        │
│                           │                             │
│              Result returned to model context           │
└─────────────────────────────────────────────────────────┘
```

Commands are triggered by the user through the UI (the terminal interface). Tools are triggered by the model through the API (tool_use content blocks in its response). A command might internally invoke a tool (for example, `/review` might use the file reading tool), but the initiation comes from the user, not the model.

This matters for permission design, for auditing, and for understanding what can be automated versus what requires human intent.

## The Registration Pattern — commands.ts

Each command is a plain object satisfying a `Command` interface:

```typescript
// commands.ts — simplified
interface Command {
  name: string                    // Without the slash: "compact", "memory"
  aliases?: string[]              // Alternative invocations
  description: string             // Shown in /help
  hidden?: boolean                // Don't show in help listing
  handler: CommandHandler
}

type CommandHandler = (
  args: string,                   // Everything after the command name
  context: CommandContext,        // Session state, message history, etc.
) => Promise<CommandResult>

interface CommandResult {
  type: 'message' | 'system' | 'noop'
  content?: string                // For 'message': show to user
  // For 'system': inject into conversation context
}
```

Registration is straightforward:

```typescript
// Registering a command
const compactCommand: Command = {
  name: 'compact',
  description: 'Summarize conversation history to free context space',
  handler: async (args, context) => {
    const summary = await compressConversation(context.messages)
    context.replaceMessages(summary)
    return {
      type: 'message',
      content: `Conversation compacted. Freed ~${summary.tokensSaved} tokens.`
    }
  }
}

// In the registry initialization
const COMMANDS: Command[] = [
  compactCommand,
  memoryCommand,
  reviewCommand,
  // ... 100+ more
]
```

The registry is a simple array. Command lookup is linear search by name (and aliases), which is fine for 100+ commands — the list is small enough that binary search or a hash map would be premature optimization.

## Dispatch — Parsing and Matching

When the user presses Enter on a slash command, the input parser checks whether it starts with `/`:

```typescript
// Dispatch logic (simplified from the terminal input handler)
function dispatch(rawInput: string, context: CommandContext): boolean {
  if (!rawInput.startsWith('/')) return false  // Not a command

  const [commandPart, ...argParts] = rawInput.slice(1).split(' ')
  const args = argParts.join(' ')

  const command = findCommand(commandPart.toLowerCase())
  if (!command) {
    showUnknownCommandError(commandPart)
    return true  // Was a command attempt, even if invalid
  }

  // Execute handler asynchronously, update UI with result
  command.handler(args, context).then(result => {
    applyCommandResult(result, context)
  })

  return true
}

function findCommand(name: string): Command | undefined {
  return COMMANDS.find(
    cmd => cmd.name === name || cmd.aliases?.includes(name)
  )
}
```

The dispatch returns `true` if it handled the input (even if the command was unknown), which tells the calling code not to treat the input as a message to send to the model.

## Notable Built-in Commands

The built-in command set covers several categories:

**Conversation management:**
```
/compact     — Summarize and compress conversation history
/clear       — Reset conversation to fresh state
/undo        — Remove last exchange from history
```

**Memory and context:**
```
/memory      — View and edit CLAUDE.md memory files
/context     — Show current context usage and budget
/cost        — Display token usage and estimated API cost
```

**Code review:**
```

```

**Session control:**
```

```

**MCP and tools:**
```

/tools       — List all available tools
```

## User-Defined Commands

One of the most powerful features is user-defined commands, which extend the built-in registry with project-specific or personal shortcuts:

```typescript
// In .claude/commands/ directory, create a .md file:
// .claude/commands/deploy.md

You are helping deploy the application.
Run: npm run build && npm run deploy
Then verify the deployment succeeded by checking /health endpoint.
```

When the user types `/deploy`, this command file is loaded, its content becomes a system prompt injection, and the model executes the instructions. This is different from the TypeScript `Command` interface — it's a prompt-based extension mechanism.

The distinction between programmatic commands (TypeScript handlers) and prompt-based commands (markdown files) means users can create powerful shortcuts without writing code. The markdown command becomes a pre-filled instruction to the model, executed with the full tool suite available.

## Commands Calling Tools Internally

Some commands invoke the tool layer beneath them:

```typescript
// /review command — internally uses file reading and diff tools
const reviewCommand: Command = {
  name: 'review',
  description: 'Review current git changes',
  handler: async (args, context) => {
    // Inject a message into the conversation asking the model to review
    // The model will then use bash/file tools to examine changes
    return {
      type: 'system',
      content: 'Please review all changes since the last git commit. ' +
               'Look for bugs, style issues, and improvement opportunities.'
    }
  }
}
```

This command doesn't directly invoke tools — it injects a system message that causes the model to invoke tools on its own. The effect is the same, but the initiation is still user-driven (the user ran `/review`), not model-driven.

## Key Takeaways

- Commands are user-initiated UI actions; tools are model-initiated API actions — the initiating party is the defining characteristic, not the implementation details.
- The `Command` interface requires only `name`, `description`, and `handler` — a minimal contract that makes adding new commands straightforward.
- User-defined commands in `.claude/commands/*.md` are prompt-based extensions that inject instructions into the model, requiring no TypeScript knowledge.
- Some commands work by injecting system messages that cause the model to use tools, maintaining the user/model boundary even when tools are ultimately involved.
- Over 100 built-in commands cover conversation management, memory, context control, and tooling — learning the command vocabulary dramatically speeds up interactive workflows.
