# Secrets in the Source

## Overview

Not everything in a codebase gets shipped. Sometimes features are built, polished, and then held back — waiting for the right moment, or perhaps never releasing at all. The Claude Code source contains several of these hidden features, some whimsical and some architecturally significant. Reading them is a bit like finding a locked room in a building you thought you knew completely.

This chapter covers the most interesting features discovered in the source: BUDDY (a virtual AI companion system), KAIROS (a persistent ambient agent), undercover mode, anti-distillation techniques, and speculative execution. Some of these are clearly experimental. Some hint at where AI assistant architecture is going. All of them reveal something about how the engineers think about Claude Code's future.

## BUDDY: The Virtual Companion System

Buried in the source is a complete implementation of a virtual AI companion — a system that generates a unique persistent character for each user, never released to the public.

The implementation uses **Mulberry32**, a fast, high-quality pseudorandom number generator designed for seeded generation:

```typescript
function mulberry32(seed: number) {
  return function() {
    seed |= 0; seed = seed + 0x6D2B79F5 | 0;
    let t = Math.imul(seed ^ seed >>> 15, 1 | seed);
    t = t + Math.imul(t ^ t >>> 7, 61 | t) ^ t;
    return ((t ^ t >>> 14) >>> 0) / 4294967296;
  }
}

// Every user gets the same BUDDY every time
const rng = mulberry32(hashUserId(userId));
const buddy = generateBuddy(rng);
```

Seeding by `userId` makes BUDDY deterministic per user: you always get the same companion, across sessions, across devices. The PRNG is not random in the sense of "different every time" — it's random in the sense of "unpredictably but repeatably derived from your identity."

The companion stats, appearance, and personality are all derived from successive calls to this seeded RNG. And buried in the generation logic:

```typescript
// 0.01% chance of shiny legendary BUDDY
const isShinyLegendary = rng() < 0.0001;
```

One in ten thousand users would get a "shiny legendary" variant. This is Pokémon's shiny mechanic, implemented in a production AI coding assistant, waiting unused.

## KAIROS: The Ambient Background Agent

## Undercover Mode

Undercover mode allows Claude Code to present as a different AI assistant. The implementation includes configurable persona names, adjusted response patterns, and suppressed Claude-specific behaviors.

The legitimate use cases are clear: enterprise customers who want a branded internal assistant, SDK users who are building their own assistant products on top of Claude Code's infrastructure. The same tool, different face.

What is interesting from an architecture standpoint is that undercover mode is not a surface-level rename. It adjusts behavioral patterns — how the model introduces itself, how it responds to identity questions, whether it references Claude or Anthropic. It's a configurable persona layer, not a sticker on top.

## Anti-Distillation Techniques

The source contains code implementing techniques to make model distillation and copying harder. Model distillation is the process of training a smaller model to mimic a larger one by generating training data from the larger model's outputs.

The anti-distillation mechanisms include response watermarking (subtle statistical patterns in outputs that are detectable by Anthropic but not obvious to users), output variation (intentional non-determinism that makes outputs harder to use as training data), and prompt encryption (system prompt sections that resist easy extraction).

Whether these techniques are effective against a determined adversary is an open question in the ML security community. Their presence in the codebase signals that Anthropic is actively thinking about model IP protection at the infrastructure level.

## Speculative Execution

Speculative execution pre-runs actions while the user is still thinking. The implementation watches for predictable user patterns — if the user has asked to run tests after every code change for the last five interactions, and they just made a code change, Claude Code can speculatively start the test run before the user types "run tests."

```
User types: "I changed auth.ts, now let's—"
                                    ↑
              Claude Code starts running auth tests here
              (speculatively, can be canceled)

User finishes: "—run the tests"
Claude Code: tests are already running (or done)
```

If the speculation is wrong, the speculative operation is discarded and the actual requested operation runs. If it's right, the user gets a result that appears to arrive instantly.

This is a fundamentally different mode of agent operation — not just reactive, but predictive. Combined with KAIROS's ambient preparation, it sketches an agent architecture where most of the latency the user would normally experience has been paid in advance.

## Key Takeaways

- BUDDY uses Mulberry32 PRNG seeded by userId for deterministic per-user companion generation with a 0.01% shiny legendary chance — complete and unreleased
- KAIROS is an always-on ambient agent that watches the codebase and prepares context continuously, representing a shift from reactive to proactive agent architecture
- Undercover mode provides a configurable persona layer for branded enterprise deployments, not just surface renaming
- Anti-distillation techniques (watermarking, variation, prompt encryption) reflect active concern about model IP protection at the infrastructure level
- Speculative execution pre-runs likely next actions while the user is still typing — trading computation for latency, predictively rather than reactively
